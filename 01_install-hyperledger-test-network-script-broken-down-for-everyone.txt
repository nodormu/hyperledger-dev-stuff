hyperledger-fabric test network install tut simplified.

Source github URLs:
https://hyperledger-fabric.readthedocs.io/en/release-2.5/install.html
https://github.com/hyperledger/fabric

Assumptions and Notes:
a) you know how to use linux CLI
b) you can see my ~/ base folder is /mnt/data/project where /mnt/data/project/fabric gets cloned into
c) you have already configured and installed docker from the official docker repo
d) I was using ubuntu 24.04 LTS server for this, but it should work on any modern linux distro, script is bash, installs are git and executables
e) assumes you understand any chowns or chmods you need to do
f) put the script in the place you want the fabric folder to get cloned into
g) why I built this? because I didn't want to have to remember how to do this all over again if I wanted to wipe it out and build up to any future point of development


Script broken down in step by step instructions
1) Clone this into your desired folder

git clone https://github.com/hyperledger/fabric.git

2) Output for my example on an externally mounted nvme drive

10) Moving forward
Now you can move on to creating applications/contracts with GO so each of the peers can "do something" to add more blocks beyond the genesis block with this git:
git clone https://github.com/hyperledger/fabric-contract-api-go.git

Be sure you change directory back out to the parent of your fabric directory for sake of your own sanity in your folder/file organization.
